# 멀티 에이전트 분석 시스템 구현 가이드

## 📋 개요

오토금융 해외진출 진단 콘솔에 멀티 에이전트 분석 시스템이 구현되었습니다.
4개의 카테고리 에이전트가 병렬로 실행되며, Summary 에이전트가 최종 통합을 수행합니다.

**주요 기능:**
- ✅ 멀티 에이전트 분석 시스템
- ✅ 실시간 진행 상태 모니터링
- ✅ 국가별 데이터 동기화
- ✅ **인터랙티브 세계 지도** (신규)

## 🎯 구현된 기능

### 1. 멀티 에이전트 구조
- **Market Agent**: 시장 카테고리 분석 (침투율, 차량가격, 시장규모)
- **Regulation Agent**: 규제 카테고리 분석 (지분한도, 라이선스, 킬스위치)
- **Environment Agent**: 금융환경 분석 (금리, LTV, 캡티브 강도)
- **System Agent**: 시스템 분석 (코어시스템, 디지털 성숙도)
- **Summary Agent**: 통합 & 최종 판정 (Phase 1 완료 후 자동 시작)

### 2. 실시간 진행 상태 UI

#### 상단 우측 상태 카드 (고정)
- **위치**: 화면 우측 상단 (모든 화면에서 표시)
- **상태**: 토글 가능 (축소/확장)
- **내용**:
  - 분석 중인 국가 (국기 + 이름)
  - 전체 진행률 프로그레스 바
  - 에이전트별 간단 상태 (✓/⏳/⏸)
  - "상세 팝업 열기" 버튼

#### 상세 진행 팝업 (모달)
- **크기**: 650px × 최대 85vh
- **내용**:
  - 전체 진행률
  - 각 에이전트별 상세 진행률 바
  - 실시간 상태 메시지
  - **Summary 에이전트 100% 완료 시에만 "보고서 보기" 버튼 활성화**
  - 소요 시간 표시

> **버튼 활성화 조건:**
> - Summary 에이전트 진행률 100% 도달 **AND** status === 'completed'
> - 그 전까지는 비활성 상태 유지 (불투명 + 클릭 불가)

### 3. 병렬 처리 시뮬레이션

```
Phase 1 (병렬):
  Market Agent    ████████████ → 100%
  Regulation      ████████████ → 100%
  Environment     ████████████ → 100%
  System          ████████████ → 100%
  
Phase 2 (순차):
  Summary Agent   ████████████ → 100%
  → 보고서 보기 활성화
```

### 4. 사용자 경험

✅ **백그라운드 분석**: 분석 중에도 다른 화면 이동 가능
✅ **실시간 업데이트**: 진행률 실시간 갱신 (800~1200ms 간격)
✅ **자동 알림**: 완료 시 상태 카드 "✓ 완료"로 전환
✅ **단계별 메시지**: 각 에이전트의 현재 작업 내용 표시
✅ **시간 측정**: 분석 소요 시간 실시간 표시

## 🚀 사용 방법

### S0 - 대시보드 (세계 지도)
1. **지도에서 국가 클릭**
   - 마우스 오버: 국가명 + 상태 툴팁 표시
   - 클릭: 우측 상세 카드 업데이트
2. **상세 카드 확인**
   - 진출국: "실적 데이터 보기" 버튼
   - 분석 가능 국가: "단일 진단 보기" 버튼
3. **버튼 클릭으로 해당 화면 이동**
   - S1 (단일 진단) 또는 S5 (데이터 관리)

### S1 - 단일 국가 진단
1. 대상 국가 선택 (예: 인도)
   - 드롭다운에서 선택 또는
   - 지도에서 국가 클릭 → "단일 진단 보기"
2. "진단 실행" 버튼 클릭
3. 우측 상단에 상태 카드 자동 표시
4. "상세보기" 또는 카드 클릭으로 상세 팝업 열기
5. 진행 상황 모니터링
6. 완료 후 "보고서 보기" 버튼 클릭
7. S4 보고서 화면으로 자동 이동

### S2 - 권역 순위
1. 권역 선택 (예: 아시아·태평양)
2. "순위 산출" 버튼 클릭
3. 권역 내 모든 국가 병렬 분석
4. 상태 카드에 "X/7 국가 완료" 표시
5. 완료 후 순위표 자동 갱신

### 상태 카드 (축소)
```
┌────────────────────────────┐

│ ████████░░ 65%  [상세보기] │

```

### 상태 카드 (확장)
```
┌────────────────────────────────┐
│ 🇮🇳 인도 분석 중           [×]│
│ ████████░░ 65%                │
│                                │
│ ✓ 시장: 완료                   │
│ ⏳ 규제: 75%                  │
│ ⏳ 환경: 40%                  │
│ ⏳ 시스템: 90%                │
│ ⏸ Summary: 대기 중            │
│                                │
│      [상세 팝업 열기]          │
└────────────────────────────────┘
```

> **개선사항:**
> - 진행률은 소수점 제외, 정수로만 표시 (65%, 75% 등)
> - 카드 높이 충분히 확보 (버튼 잘림 방지)
> - 분석 실행 시 팝업 자동 열림 제거 (사용자가 직접 "상세 팝업 열기" 클릭)

> - 진행률 100% 도달 시 자동으로 🔄 → ✓ 전환

## 🔧 기술 구현

### JavaScript 상태 관리
```javascript
analysisState = {
  isRunning: boolean,
  country: string,           // 선택된 국가명 (예: "영국")
  countryCode: string,       // 선택된 국가 코드 (예: "uk")
  countryFlag: emoji,        // 국가 플래그 이모지
  startTime: timestamp,
  agents: {
    market: { progress: 0-100, status: 'waiting'|'running'|'completed', message: string },
    regulation: { ... },
    environment: { ... },
    system: { ... },
    summary: { ... }
  }
}
```

### 국가 데이터 동기화
```javascript
// 1. 사용자가 S1에서 국가 선택
targetCountry = 'uk'  // 영국

// 2. 진단 실행 시 전역 상태에 저장
analysisState.countryCode = 'uk'
analysisState.country = '영국'
analysisState.countryFlag = '🇬🇧'

// 3. 모든 UI 컴포넌트가 동일한 상태 참조
- 상태 카드: analysisState.country 사용
- 결과 패널: countryData[analysisState.countryCode] 사용
- 보고서: analysisState.countryCode 기반 데이터 표시

// 4. 국가 변경 시 모든 UI 자동 업데이트
updateReportView() → 선택된 국가 데이터로 보고서 갱신
```

## 🎬 데모 시나리오

1. **브라우저에서 dashboard.html 열기**
2. **좌측 네비게이션 → "단일 국가 진단" 클릭**
3. **대상 국가: 인도 선택**
4. **"진단 실행" 버튼 클릭**
5. **우측 상단 상태 카드 자동 표시 확인 (확장 상태)**
6. **"상세 팝업 열기" 버튼 클릭 (선택사항)**
7. **각 에이전트 진행률 실시간 업데이트 확인**
   - 진행 중: 🔄 스피너 아이콘 + 진행률 (정수)
   - 완료: ✓ 체크 아이콘 (초록색)
8. **Phase 1 (4개 에이전트) 완료 후 Summary 에이전트 자동 시작**
9. **Summary 에이전트 100% 완료**
   - Summary도 ✓ 아이콘으로 전환 확인
   - "보고서 보기" 버튼 즉시 활성화
10. **버튼 클릭 → S4 보고서 화면으로 이동**

> **주요 변경사항:**
> - 팝업 자동 열림 제거 → 사용자가 직접 "상세 팝업 열기" 클릭
> - 진행률 정수 표시 (65%, 75% - 소수점 제거)
> - **모든 에이전트 (Summary 포함) 완료 시 ✓ 아이콘으로 표시**
> - **"보고서 보기" 버튼은 Summary 100% 완료 시에만 활성화**
> - 카드 레이아웃 개선 (버튼 잘림 방지)
