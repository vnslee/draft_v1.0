# 고객 세그먼트 (Customer Segment)

> **목적** — '글로벌 진출 전략 시뮬레이션 AGENT'에서 **특정 국가가 입력되면 해당 국가의 자동차 / 자동차 금융 고객 세그먼트**를 제시할 수 있도록, 세그먼트 **분류 체계(Taxonomy)**와 **국가별 수집 항목**을 정의한다.

이 문서는 두 가지를 함께 정의한다.
1. **분류 체계** — 자동차 / 자동차 금융 고객을 어떤 축(Axis)으로 나눌지에 대한 **표준 구분**
2. **수집 정의** — 위 분류 체계를 기준으로 **국가별로 어떤 데이터를 수집**할지

> 분류 체계는 **모든 국가에 공통 적용되는 틀**이고, 수집 데이터는 **국가별로 채워지는 값**이다.

---

## 1. 대상

**대상 국가**별로, 아래 두 영역의 고객 세그먼트를 수집한다.

| 영역 | 설명 |
|------|------|
| 자동차 (Vehicle) | 차량을 **구매·보유**하는 고객의 세그먼트 |
| 자동차 금융 (Auto Finance) | 차량을 **금융 상품(할부·리스·렌트 등)으로 이용**하는 고객의 세그먼트 |

> 한 고객은 두 영역에 동시에 속할 수 있다. (예: "30대 첫차 구매 고객"이면서 "할부 금융 이용 고객")

---

## 2. 분류 체계 (Taxonomy)

세그먼트는 **단일 기준이 아니라 여러 축의 조합**으로 정의한다.
국가별 데이터가 확보되는 축만 채우고, **미확보 축은 비워 둔다.**

> ⚠️ **분류 구분값은 국가별로 다를 수 있다.** 차급·채널 등은 국가마다 통용되는 분류 자체가 다르므로(아래 비고 참조), 구분 예시는 **참고용**이며 실제 수집 시에는 **현지 기준 구분값**을 쓴다.

### 2-1. 자동차 (Vehicle) 세그먼트 축

| 축 (Axis) | 구분 예시 | 비고 |
|-----------|-----------|------|
| 차급 (Segment Class) | 경형 / 소형 / 준중형 / 중형 / 대형 / SUV / 상용 | **국가별 차급 분류 체계가 다름.** 예: 미국은 세단보다 유틸리티·픽업 중심으로 세분(컴팩트 유틸리티 / 어퍼미드사이즈 유틸리티 / 풀사이즈 하프톤 픽업 등). 현지 기준을 우선하고 `note`에 분류 차이를 기록 |
| 가격대 (Price Tier) | 엔트리 / 볼륨 / 프리미엄 / 럭셔리 | 현지 통화 기준 가격 구간 |
| 브랜드 성격 (Brand Origin) | 로컬 / 수입 / 프리미엄 수입 | — |
| 구매 연령대 (Age) | 2030 / 40대 / 5060+ | 주요 구매층 |
| 용도 (Usage) | 개인(자가) / 법인·리스 / 영업·상용 | — |
| 동력원 (Powertrain) | 내연기관(ICE) / 하이브리드 / 전기(EV) | 국가 전동화 수준 반영 |

### 2-2. 자동차 금융 (Auto Finance) 세그먼트 축

| 축 (Axis) | 구분 예시 | 비고 |
|-----------|-----------|------|
| 금융 상품 (Product) | 할부(오토론) / 리스 / 장기렌트 / 일시불 | 국가별 통용 상품 유형 |
| 차량 상태 (Vehicle State) | 신차 / 중고차 | — |
| 채널 (Channel) | 캡티브(제조사 금융) / 은행 / 카드사 / 전업 캐피탈 / 크레딧유니온 | **국가별 금융 채널 구성이 다름.** 예: 한국은 카드사 오토할부 채널이 있으나 미국은 없고, 대신 크레딧유니온(약 20%)이 주요 채널. 해당 국가에 없는 채널은 수집 대상에서 제외 |
| 신용 등급 (Credit) | 프라임 / 준프라임 / 서브프라임 | 국가 신용평가 체계 기준 |
| 고객 유형 (Customer Type) | 개인 / 법인 | — |
| 약정 구조 (Term) | 단기 / 중기 / 장기, 잔가보장 여부 | 확보 시에만 |

## 3. 수집 정보

각 국가별로, **위 분류 체계의 각 축에 대해** 아래 정보를 수집한다.

| 구분 | 항목 | 설명 |
|------|------|------|
| 식별 | 국가명 / 권역 | 국가 식별 및 분류 기준 |
| 식별 | 영역 | 자동차 / 자동차 금융 |
| 세그먼트 | 축 (Axis) | 2장에서 정의한 축 |
| 세그먼트 | 구분값 (Segment) | 해당 축 내 세부 세그먼트 (예: "프리미엄 수입") |
| 규모 | 비중(share) | 해당 세그먼트가 시장에서 차지하는 비중 (%) — **출처에 명시된 경우만** |
| 규모 | **측정 기준(basis)** | share의 **분모**. 판매(sales) / 신규등록(registrations) / 금융취급(financing) / 보유대수(parc) 등 — **출처가 어떤 기준으로 집계했는지 반드시 기록** |
| 규모 | 성장성(trend) | 증가 / 유지 / 감소 — 확보 시에만 |
| 특성 | 주요 특징(note) | 대표 구매 동기·선호 등 정성 정보 |

> **차급·점유율 등 정량 수치는 추정 금지.** 출처가 있는 값만 기재하고, 없으면 비워 둔다. (누락 = 미확보)
> **`basis`(측정 기준)는 필수.** 같은 항목도 출처에 따라 분모가 다르다(예: 미국 EV 비중 = 판매 기준 7.8% vs 신규등록 기준 7.4%). 분모가 다른 값은 **같은 항목이 아니므로 합산·비교하지 않는다.**

---

## 4. 제약 사항

1. **분류 체계(2장)는 공통 틀(축)**이며, 국가별로 통용되는 **차급·금융 채널·금융 상품·신용 등급의 구분값이 다를 경우 현지 기준을 우선**하고 `note`에 차이를 기록한다. (예: 미국 차급은 유틸리티·픽업 중심 세분 / 미국 금융 채널은 카드사 없이 크레딧유니온 존재)
2. **점유율·비중 등 정량 데이터는 출처에 명시된 경우만 기재**한다. 추정·통념으로 채우지 않는다. (누락 = 미확보)
3. 모든 세그먼트 항목에는 **출처·데이터 기준연도·측정 기준(basis)**을 명기한다. (기본 기준연도: 2025년)
4. **분모(basis)·출처·시점이 다른 share는 합산하거나 직접 비교하지 않는다.** (예: 판매 기준과 등록 기준, Q1과 Q4를 섞지 않는다)

---

## 5. 산출물 생성 프로세스

- 조사한 세그먼트 정보를 **JSON 초안**으로 생성한다.
- **산출물 경로**: `mockup/customer_segment/` 폴더에 생성한다. (폴더가 없으면 생성)
- JSON 구조는 **초안 수준**으로 둔다.
- **미확보 데이터는 채워넣지 않는다** (누락 = 미확보).
- 신뢰성 정보가 필요한 데이터 필드는 **`{ value, year, estimated, source, basis?, note? }`** 메타 구조로 저장한다.
    - `estimated`: 추정치/미확정이면 `true`, 확정 실측치면 `false`
    - `basis`: share의 측정 기준(분모). 판매 / 신규등록 / 금융취급 / 보유대수 등. 정량 share에는 가급적 함께 기록한다.
    - agent가 표시 시점에 **연도 / 추정 여부 / 출처 / 측정 기준**을 함께 보여줄 수 있도록 한다.
- 각 **축(axis) 단위로 `basis`**를 함께 둘 수 있다(해당 축의 items가 같은 기준이면 축 레벨에 1회 표기).
- 세그먼트 목록처럼 전체 분류를 다 채웠는지 여부는 **`complete`** 플래그로 목록 완성 여부를 표기한다.
