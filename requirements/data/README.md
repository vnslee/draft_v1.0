# 데이터 요구사항 (Data Requirements)

> **목적** — '글로벌 진출 전략 시뮬레이션 AGENT'에 들어갈 **초기 데이터**의 수집 범위·구조·제약 사항을 정의한다.

이 폴더는 AGENT가 시뮬레이션을 수행하기 위해 필요한 **기초 데이터(Seed Data)**의 요구사항을 다룬다.
시스템 개발 요구사항(DB 스키마, 기술 스택, 아키텍처)은 [`../agent/`](../agent/) 폴더를 참고한다.

---

## 1. 폴더 구조

| 폴더 | 구분 | 설명 |
|------|------|------|
| [`00_basic_information/`](00_basic_information/) | 기본 정보 | 대상 국가, 인구, 자동차 시장 기초 정보 |
| [`01_market_landscape/`](01_market_landscape/) | 시장 환경 | 브랜드 / OEM / 모델 등 시장 구도 및 국가 간 유사도 |
| [`02_regulations/`](02_regulations/) | 규제 | 국가별 진출 관련 규제·인증·관세 등 |
| [`09_schema/`](09_schema/) | 스키마 | 수집 데이터의 구조 정의 / 데이터 모델 초안 |

> 폴더 번호는 데이터의 **논리적 처리 순서**를 의미한다. (기초 정보 → 시장 환경 → 규제 → 스키마화)

---

## 2. 공통 데이터 정책

모든 데이터 수집은 아래 정책을 따른다.

1. **제약 준수** — [`global_constraints.md`](global_constraints.md)에 정의된 기본 제약 사항을 반드시 준수한다.
2. **출처 명기** — 모든 조사 항목에는 반드시 출처와 데이터 기준연도를 함께 기록한다.
3. **지어내기 금지** — 미확보 데이터는 채워넣지 않는다. (누락 = 미확보)
4. **기준 연도** — 별도 명시가 없는 한 **2025년**을 기준으로 한다.

---

## 3. 산출물 (Output)

- 조사한 데이터는 **JSON 초안**으로 생성한다.
- 산출물에는 **데이터 기준연도**와 **출처**를 메타 정보로 함께 기록한다.
- JSON 구조는 **초안 수준**으로 두고, DB 스키마([`09_schema/`](09_schema/))가 확정되면 정규화·테이블 구조에 맞춰 조정한다.
- 각 영역별 산출물 경로는 해당 폴더의 README 또는 정의 문서를 따른다.

---

## 4. 작성 가이드

각 하위 폴더의 데이터 정의 문서는 아래 구조를 권장한다.

```
# <데이터 영역명>
> 목적 — 이 데이터가 AGENT에서 어떤 역할을 하는지

## 1. 대상      ← 무엇을 수집하는가
## 2. 수집 정보  ← 어떤 항목을 수집하는가 (표 형태)
## 3. 제약 사항  ← global_constraints 외 추가 제약
## 4. 산출물     ← 결과물 형태 / 경로 / 출처 기록 방식
```
