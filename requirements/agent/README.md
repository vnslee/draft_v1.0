# 에이전트 시스템 요구사항 (Agent System Requirements)

> **목적** — '글로벌 진출 전략 시뮬레이션 AGENT'를 실제로 **개발·구축**하기 위한 시스템 요구사항을 정의한다.

이 폴더는 AGENT의 **시스템 개발 요구사항**(DB 스키마, 기술 스택, 아키텍처 구조 등)을 다룬다.
AGENT에 투입되는 초기 데이터의 요구사항은 [`../data/`](../data/) 폴더를 참고한다.

---

## 1. 폴더 구조 (계획)

| 폴더 / 문서 | 구분 | 설명 |
|-------------|------|------|
| `01_architecture/` | 아키텍처 | 시스템 구성도, 에이전트 흐름(Flow), 컴포넌트 책임 정의 |
| `02_tech_stack/` | 기술 스택 | 언어 / 프레임워크 / LLM / 인프라 등 사용 기술 정의 |
| `03_db_schema/` | DB 스키마 | 테이블 정의, ERD, 데이터 모델 ([`../data/09_schema/`](../data/09_schema/)와 연계) |
| `04_agent_design/` | 에이전트 설계 | 역할(Role), 도구(Tool), 프롬프트, 추론 흐름 정의 |
| `05_api_interface/` | 인터페이스 | 외부 입출력 API / 데이터 연동 규격 |

> 폴더는 요구사항 정의가 구체화되는 시점에 생성한다. 현재는 **계획안(Skeleton)**이다.

---

## 2. 정의 대상 요구사항

시스템 개발을 위해 아래 항목을 정의한다.

| 구분 | 항목 | 설명 |
|------|------|------|
| 아키텍처 | 시스템 구조 | 전체 컴포넌트 구성 및 데이터 흐름 |
| 아키텍처 | 에이전트 구조 | 단일 / 멀티 에이전트 여부, 오케스트레이션 방식 |
| 기술 스택 | 언어 / 프레임워크 | 백엔드·프론트엔드·에이전트 프레임워크 |
| 기술 스택 | LLM | 사용 모델, 호출 방식, 비용·성능 고려 사항 |
| 기술 스택 | 인프라 | 실행 환경, 배포, 스토리지 |
| 데이터 | DB 스키마 | 테이블 / 관계 / 인덱스 정의 |
| 데이터 | 데이터 소스 | 초기 데이터([`../data/`](../data/)) 적재·갱신 방식 |
| 에이전트 | 역할 / 도구 | 에이전트별 책임, 사용 도구(Tool), 권한 |
| 에이전트 | 추론 흐름 | 입력 → 추론 → 시뮬레이션 → 출력 파이프라인 |
| 인터페이스 | 입출력 규격 | 사용자 요청 / 시뮬레이션 결과 포맷 |

---

## 3. 설계 원칙

1. **데이터 정합성** — DB 스키마는 [`../data/`](../data/)에서 정의한 수집 데이터 구조와 일관되게 설계한다.
2. **확장성** — 진출 대상 국가·데이터 영역이 추가되어도 구조 변경이 최소화되도록 설계한다.
3. **추적 가능성** — 시뮬레이션 결과는 근거 데이터와 출처를 추적할 수 있어야 한다.
4. **명세 우선** — 구현 전 각 요구사항을 문서로 정의한 뒤 개발에 착수한다.

---

## 4. 작성 가이드

각 시스템 요구사항 문서는 아래 구조를 권장한다.

```
# <요구사항 영역명>
> 목적 — 이 요구사항이 시스템에서 차지하는 역할

## 1. 개요       ← 무엇을 정의하는가
## 2. 상세 요구사항 ← 구조 / 스키마 / 스택 등 (표·다이어그램)
## 3. 제약 사항    ← 기술적·운영적 제약
## 4. 연관 항목    ← 연계되는 data / 다른 요구사항 문서
```
