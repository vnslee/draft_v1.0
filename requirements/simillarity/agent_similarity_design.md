# 글로벌 진출 유사도 평가 AGENT — 설계문서

> 작성일: 2026-06-14
> 상태: 설계 초안 (방식 검증 3건 완료, baseline 목업 적용, DB 스키마 확정 전)

---

## 1. 목적

진출국(이미 진출한 나라)과 **진출 예정국**을 비교하여:

1. 두 국가의 **규제·시장 유사도**를 0~1 점수로 산출
2. 그 유사도를 근거로 **진출 소요기간·비용**을 추정
3. **"A국보다 B국이 진출에 유리"** 같은 의사결정을 보고서로 출력

핵심 아이디어: **같은 권역(region)의 기준국가가 실제로 쓴 비용/기간**을 baseline으로 두고,
후보국과 기준국의 유사도로 **배수(multiplier)**를 매겨 추정한다.

> 예) 미국 진출에 실제 12개월 소요. 멕시코는 미국과 같은 미주 권역이고 유사도 0.8 →
> 배수 ×1.2 → 멕시코 약 14.4개월 예상.

## 2. 입력 데이터

`mockup/` 폴더의 JSON. **비교 대상 데이터셋**과 **참조용 데이터셋**을 구분한다.

| 파일 | 역할 | 비교에 사용? |
|---|---|---|
| `regulations/auto_finance_regulation.json` | 자동차 금융 규제·세제 | ✅ 비교 차원 |
| `regulations/capital_license.json` | 캐피탈 진출 라이선스 | ✅ 비교 차원 |
| `purchase_process/purchase_process.json` | 자동차 구매 프로세스 | ✅ 비교 차원 |
| `customer_segment/customer_segment.json` | 고객 세그먼트 | ✅ 비교 차원 |
| `countries/countries.json` | 국가 정보(인구·GDP·판매·권역·진출상태) | ⚙️ 참조(권역·후보선정 메타) — **항목 유사도 비교 대상 아님** |
| `baseline/entry_baseline.json` | 권역별 진출 실측 기간/비용 | ⚙️ 참조(소요기간 추정 baseline) — **현재 목업** |

> 사용자 지시: `countries`/`customer_segment` 폴더의 **국가·세그먼트 정의 자체**는 유사도 비교 대상에서 제외.
> 단 `countries.json`의 `region`, `entry_status`, GDP 등은 권역 매칭·후보 선정의 **참조 메타**로 활용.

### 2.1 데이터 공통 특성

- 모든 값 필드가 신뢰성 메타 객체: `{value, year, estimated, source, quote?, note?, status?}`
- `value: null` 또는 `status: "공식 자료에서 확인 불가"` = **미확보**(누락 ≠ 0, 지어내기 금지)
- `estimated: true` = 추정/미확정, `false` = 공식출처 확정
- **데이터셋마다 내부 구조가 다름**:
  - `auto_finance_regulation`: 국가 → 상품(auto_loan/ev_loan/lease…) → category → 하위 fact (비대칭, 단위 제각각)
  - `capital_license`: 국가 → license/authority/legal_basis/capital_requirement/procedure
  - `purchase_process`: 국가 → dimension(4개 고정 축) → attributes[] **(이미 유사도 비교용으로 설계됨)**


## 4. 1층 — 항목 유사도 엔진

같은 `(데이터셋, 상품/축, category/aspect)`를 가진 두 국가 fact를 짝지어 점수화.

### 4.1 4축 점수

| 축 | 보는 것 | 계산 | 미확보 처리 |
|---|---|---|---|
| **존재성 (presence)** | 규제 있음/없음(확인)/모름 | `is_missing`, value 유무 조합을 규칙 매핑 | 한쪽 미확보 → 존재성 자체가 신호 |
| **성격 (stance)** | 상한있음/없음/금지/공시/위임 등 *방향* | `value_text`에서 stance 라벨 추출 후 일치도 | — |
| **수치 근접 (magnitude)** | 숫자가 얼마나 가까운가 | 단위·통화 정규화 후 `1 − |a−b|/max(a,b)` | 한쪽 null → **N/A(마스킹)**, 점수에서 제외 후 재정규화 |
| **의미 (semantic)** | 서술 문장의 의미 유사 | `value_text`+`quote` 임베딩 코사인 | — |

**기본 가중치(조정 가능)**: 존재성 0.35 / 성격 0.30 / 수치 0.20 / 의미 0.15

> 항목에 수치가 없으면 수치 축을 빼고 나머지 3축을 재정규화한다.

### 4.2 핵심 규칙 (검증으로 확인된 것)

- **미확보(null) ≠ 0점**: "다름"이 아니라 "모름". 수치 축에서 마스킹하고, 보고서에 "비교 불가"로 명시.
- **"둘 다 미확보"는 유사도 높음이 아니라 N/A**: 따로 분리 집계.
- **단위/통화 정규화 필수**: $10,000 vs ₹150,000을 그대로 빼면 무의미.
  통화는 환율 기준(명목 vs PPP)을 §6 정규화 사전에 고정.
- **하위항목 개수 비대칭(예: 미국 5개 vs 인도 3개)**: key 1:1 매칭 불가 →
  공통 **aspect**(예: "금리상한", "중도상환")로 재그룹핑 후 비교.

### 4.3 점수 vs 근거 역할 분리

- **점수는 규칙(코드)으로** 산출 → 재현 가능·결정적.
- **근거 문장은 LLM(Claude)으로** 생성 → 양국 fact를 주고 "왜 이 점수인지" 서술.

## 5. 2층 — 데이터셋별 유사도 집계

각 데이터셋 내 모든 항목 점수를 모아 데이터셋 단위 점수로 집계.

- 항목별 점수를 **가중 평균** (중요 항목 가중치 부여 가능; 예: license의 capital_requirement↑)
- **미확보 항목은 분모에서 제외**하고 `coverage`(확보율)를 함께 보고.
  → 점수 0.7이라도 coverage 40%면 신뢰도 낮음을 표시.
- 데이터셋별로 **공통 aspect 사전**(§6)에 정의된 축만 비교(축이 다르면 비교 불성립).

출력 예:
```
license:   0.78  (coverage 80%, 4/5 항목 확보)
regulation:0.72  (coverage 90%)
purchase:  0.81  (coverage 100%)
segment:   0.65  (coverage 70%)
```

## 6. 공통 aspect / 정규화 사전 (10개국 확장의 토대)

데이터셋마다 별도 정의. 국가가 늘어도 이 사전만 유지하면 비교가 성립.

### 6.1 aspect 사전 (데이터셋별 비교축 고정)

| 데이터셋 | 공통 aspect (예시) |
|---|---|
| capital_license | license_structure, authority, capital_amount, procedure_duration |
| auto_finance_regulation | term_limit, ltv, rate_cap, prepayment, tax_deduction, gst/sales_tax |
| purchase_process | online_digital, dealership_model, financing_incentives, ev_process (이미 정의됨) |
| customer_segment | (domain=vehicle/auto_finance 별 축 — 데이터 추가확인 필요) |

### 6.2 stance 라벨 사전 (성격 축 분류값)

```
규제 존재 방향: { 상한있음, 상한없음(확인), 금지, 공시의무, 위임, 면제, 미확보 }
세제 방향:      { 공제있음, 공제없음, 감면, 종료, 미확보 }
```

### 6.3 단위·통화 정규화 규칙

| 단위 종류 | 정규화 기준 |
|---|---|
| 비율(%) | 소수(0.16)로 통일 |
| 통화 | **기준통화 1개로 환산**(예: USD). 환율표는 별도 관리, 기준연도 명시. PPP 적용 여부는 옵션 플래그 |
| 기간 | 개월(months)로 통일 |
| 금액 lakh/crore | crore=1e7, lakh=1e5 변환 후 통화 환산 |

> ⚠️ 통화 환산 기준(명목 vs PPP, 환율 기준연도)은 점수를 크게 좌우 → 보고서에 항상 **가정으로 명시**.

### 6.4 권역(region) 정규화

현재 세 파일(`countries`, `capital_license`, `auto_finance_regulation`)의 권역 표기는
`{미주, 유럽, 아시아 & 태평양}`으로 **통일됨**(2026-06-14 오타 수정 완료).
→ 다만 향후 데이터셋 추가 시 표기 불일치를 막기 위해 권역 매핑 테이블을 안전장치로 유지:
   `{미주, 유럽, 아시아 & 태평양, (아시아/아태/APAC 등 변형 → 아시아 & 태평양) …}`.
baseline 국가 선택 시 동일 권역으로 묶기 위해 필수.

## 7. 3층 — 의사결정 엔진

### 7.1 판단 논리: **유사도 + 절대난이도 혼합 (논리 B)**

> 검증 결과: 유사도 단일(논리 A)은 "인도 자본금 ₹10cr이 €5M의 1/4로 낮다"는
> *유리한* 정보를 "독일과 다름"으로 잘못 감점함. 진출 유리함 판단에는 절대난이도가 필요.

```
종합점수 = 유사도 × w_sim + (1 − 절대난이도) × w_diff      (예: w_sim=w_diff=0.5)
```

- **유사도**: 2층 데이터셋별 점수의 가중 종합
- **절대난이도**: 자본금 절대액·절차 소요기간 등 "진입장벽 크기 자체"
  (낮을수록 진출 쉬움; 데이터 있을 때만, 없으면 등급으로)

### 7.2 진출 소요기간·비용 추정 (권역 baseline 배수 모델)

**핵심 모델**: 같은 권역의 기준국가 실측치 × 유사도 기반 배수.

```
추정기간 = baseline_기간 × multiplier(유사도)
추정비용 = baseline_비용 × multiplier(유사도)
```

- `baseline`: 같은 권역에서 이미 진출한 국가의 **실측 기간/비용**
  - **MVP 현재: 목업으로 채워짐** → `mockup/baseline/entry_baseline.json`
    (권역 대표: 미주=미국, 유럽=독일, 아태=한국). 실측 교체 방법은 `mockup/baseline/README.md` 참조.
  - 실데이터 확보 시 `duration_months`/`cost_usd` 값과 `multiplier_table`만 교체하면 됨.
- `multiplier`: 유사도 구간별 가감 기준표 — **`entry_baseline.json`의 `multiplier_table`에 정의(현재 목업)**

| 유사도 구간 | 배수(기간) | 해석 |
|---|---|---|
| 0.9 ~ 1.0 | ×1.0 | 거의 동일 — baseline 그대로 |
| 0.8 ~ 0.9 | ×1.2 | 약간 차이 — 20% 추가 |
| 0.6 ~ 0.8 | ×1.5 | 보통 차이 |
| 0.4 ~ 0.6 | ×2.0 | 상당 차이 |
| < 0.4 | ×2.5+ 또는 "별도 조사 필요" | 기준국 경험 재사용 한계 |

> 사용자 예시: "미국 12개월, 멕시코 유사도 0.8 → ×1.2 → 약 14.4개월".
> 배수 테이블은 가설(목업) — 실제 진출 데이터로 보정 예정.

### 7.3 출력: A vs B 비교 판단

```
후보 순위 = 종합점수 내림차순
각 후보: 추정기간, 추정비용, 종합점수, coverage, 핵심 근거
```

| 항목 | MVP 결정 | 추후 업그레이드 |
