> 버전 v0.2 · 최종 수정일 2026-06-16 · 용도: 화면 간 이동 구조(IA) + 진입/전환 정의 + 멀티 에이전트 분석 프로세스
> 선행 문서: 01 설계문서 9장(화면 목록), 4장(기능 명세)

**변경 이력:**
- v0.1 (2026-06-14): 초안 작성 - 기본 화면 흐름 및 이동 정의
- v0.2 (2026-06-16): 멀티 에이전트 분석 아키텍처 추가

  - 사용자 여정 업데이트 (에이전트 진행 과정 포함)
  - 구현 가이드 및 데이터 소스 추가

## 0. 읽는 법

- 본 문서는 **화면 간 이동 구조**만 정의한다. 개별 화면의 내부 레이아웃·컴포넌트는 다음 문서(05 화면설계_상세)에서 다룬다.
- 화면 코드: `S0`(메인) ~ `S5`(데이터 관리). 이후 문서·프로토타입에서 동일 코드 사용.

## 1. 화면 목록 (01 설계서 9장과 매핑)

| 코드 | 화면 | 역할 | 연결 기능 |
|---|---|---|---|
| S0 | 메인/대시보드 | 진입점, 기능 선택, 최근 분석 요약 | - |
| S1 | 단일 국가 진단 | 국가 입력 → 유사도 결과 + 판정 + 비용 | 기능1 |
| S2 | 권역 순위 | 권역 선택 → 순위표(카테고리 분해) | 기능2 |
| S3 | 설정 | 가중치/임계값/게이트/킬스위치/화이트리스트 | 서브(B) |
| S4 | 보고서 뷰어 | 요약/AI인사이트/상세/출처/조사일자/가중치 | 서브(A) |
| S5 | 데이터 관리 | 진출국 실적 입력, 조사 데이터 갱신·이력 | 데이터 자산 |

## 3. 진입·전환 정의

### 3.1 S0 메인 → 각 화면

| 트리거 | 이동 | 비고 |
|---|---|---|
| "국가 진단" 선택 | S0 → S1 | 기능1 진입 |
| "권역 순위" 선택 | S0 → S2 | 기능2 진입 |
| "설정" 선택 | S0 → S3 | |
| "데이터 관리" 선택 | S0 → S5 | |
| 최근 보고서 카드 클릭 | S0 → S4 | 저장된 보고서 바로 열람 |
| **지도 내 국가 클릭** | **국가 상세 카드 갱신** | 선택된 국가 정보 표시, 상태별 액션 버튼 |
| **상세 카드 "단일 진단 보기"** | **S0 → S1** | 해당 국가 선택 상태로 이동 |

> **세계 지도 기능:**
> - SVG 기반 인터랙티브 지도 (주요 국가만 표시)
> - 국가별 색상 구분: 진출국(검정) / 이식가능(초록) / 심층조사(황색) / 차단(적색) / 미조사(회색)
> - 마우스 오버 시 툴팁 표시 (국가명 + 상태)
> - 클릭 시 우측 상세 카드 업데이트

### 3.2 S1 단일 국가 진단

| 트리거 | 이동 | 처리 |
|---|---|---|
| 국가 입력 + 분석 실행 | S1 내 결과 표시 | 기능1 STEP 1~4 수행 → **멀티 에이전트 분석 시작** |
| 분석 실행 중 | 상단 우측 분석 상태 카드 표시 | **선택한 국가명** + 진행률 표시 |
| "상세보기" 클릭 | 분석 진행 팝업 모달 표시 | 각 에이전트별 상세 진행률 표시 |
| 분석 완료 | 팝업 내 "보고서 보기" 버튼 활성화 | **선택한 국가 데이터로 결과 표시** |
| 판정=이식가능/심층조사 | S1 결과 → "보고서 보기" | |
| "보고서 보기" | S1 → S4 | **선택한 국가 기반** 보고서 생성·열람 |
| "가중치 바꿔 재실행" | S1 → S3(설정) → 복귀 | 또는 S1 내 간이 가중치 패널 |
| 데이터 부족 경고 | S1 → S5 | 조사 데이터 보완 유도 |

> **분석 실행 전 검증**: 대상국 조사 데이터(RESEARCH_SNAPSHOT)가 없으면 S5로 유도. 같은 권역 진출국이 없으면 "비교 대상 없음" 안내.

> **데이터 동기화 원칙:**
> - 사용자가 S1에서 선택한 국가 정보는 전역 상태(analysisState)에 저장
> - 상태 카드, 팝업, 결과 패널, 보고서 모두 동일한 국가 데이터 참조
> - 국가 변경 시 모든 UI 컴포넌트 동시 업데이트
> - 하드코딩된 국가 데이터 사용 금지 (Mock 데이터도 선택한 국가 기반으로 생성)

#### 3.2.1 멀티 에이전트 분석 프로세스 (신규)

**분석 실행 버튼 클릭 시 작동 순서:**

1. **분석 시작 → 상단 우측 상태 카드 표시**
   - 위치: 화면 최상단 우측 (룰셋 칩 하단 또는 인접)
   - 표시 형태: 토글 가능한 카드
   - 내용: "🇮🇳 인도 분석 중... 전체 진행률 45%"
   - 상태: 클릭 시 상세 팝업으로 확장

2. **병렬 에이전트 실행 (4개 에이전트 동시 작동)**
   ```
   ┌─ 에이전트 A: 시장 분석 (Market Analysis)
   ├─ 에이전트 B: 규제 분석 (Regulation Analysis)  
   ├─ 에이전트 C: 환경 분석 (Financial Environment)
   └─ 에이전트 D: 시스템 분석 (System Analysis)
   ```
   - 각 에이전트는 독립적으로 데이터 리서치 및 분석 수행
   - 실시간 진행률 업데이트 (0% → 100%)

3. **통합 단계 (모든 에이전트 완료 후)**
   - 에이전트 E: 데이터 취합 & Summary 작성
   - 카테고리별 점수 계산
   - 킬스위치·게이트 검증
   - 최종 판정 및 비용 추정

4. **완료 상태**
   - 상단 카드: "✓ 인도 분석 완료"
   - 팝업 하단: "보고서 보기" 버튼 활성화
   - 버튼 클릭 시 S4(보고서 뷰어)로 이동

### 3.3 S2 권역 순위

| 트리거 | 이동 | 처리 |
|---|---|---|
| 권역 선택 + 실행 | S2 내 순위표 | 권역 내 전 국가에 기능1 루프 → **멀티 에이전트 병렬 분석** |
| 분석 실행 중 | 상단 우측 분석 상태 카드 표시 | "유럽 권역 분석 중 (3/7 국가 완료)" |
| "상세보기" 클릭 | 분석 진행 팝업 모달 표시 | 국가별 에이전트 진행률 표시 |
| 순위표 행 클릭 | S2 → S1(해당국 상세) 또는 S4 | 개별 국가 드릴다운 |
| "권역 보고서" | S2 → S4 | 권역 종합 보고서 |

> **권역 분석 특성**: 권역 내 모든 대상국(예: 유럽 7개국)에 대해 동시 분석 수행. 각 국가별로 4개 에이전트가 작동하므로 총 28개(7×4) 에이전트가 병렬 실행됨.

### 3.4 S3 설정 → 결과 재계산

| 트리거 | 이동 | 처리 |
|---|---|---|
| 가중치/임계값 저장 | S3 내 저장 | **새 ruleset 버전 생성**(기존 수정 금지) |
| "저장 후 재분석" | S3 → S1 또는 S2 | 직전 분석을 새 룰셋으로 재실행 |

> **재현성 주의**: 설정 변경은 새 ruleset을 만든다. 과거 보고서(S4)는 그때의 weight_snapshot으로 그대로 재현된다(설계서 8장 리스크7).

### 3.5 S4 보고서 뷰어

| 트리거 | 이동 | 비고 |
|---|---|---|
| "원본 분석으로" | S4 → S1/S2 | 분석 화면 복귀 |
| "이 보고서 가중치로 재실행" | S4 → S3 | 보고서의 weight_snapshot을 설정에 로드 |
| 출처 항목 클릭 | S4 내 출처 패널 | 공식/Tier 표시 확인 |

### 3.6 S5 데이터 관리

| 트리거 | 이동 | 처리 |
|---|---|---|
| 진출국 실적 입력·수정 | S5 내 | ENTRY_RECORD 갱신 |
| 조사 데이터 갱신 | S5 내 | 새 RESEARCH_SNAPSHOT 생성(조사일자) |
| "이 국가 진단하기" | S5 → S1 | 조사 직후 바로 분석 |

## 4. 핵심 사용자 여정 (User Journey)

### 여정 A — "이 나라 진출하면 얼마 들까?" (기능1 중심, 멀티 에이전트 포함)
```
S0 → S1(체코 입력) → [분석 실행] 
   → 상단 우측 "체코 분석 중" 카드 표시
   → [상세보기 클릭] → 에이전트 진행 팝업
      • 시장 분석: 100% ✓
      • 규제 분석: 85% ⏳
      • 환경 분석: 70% ⏳
      • 시스템 분석: 95% ⏳
      • Summary: 대기 중
   → [약 1분 후 완료]
   → 팝업 내 "보고서 보기" 버튼 활성화
   → S4(근거 보고서 확인)
   → [가중치 의심] → S3(시스템 비중↑)
   → S1 재실행 → [멀티 에이전트 재분석] → S4(재보고서)
```

### 여정 B — "어느 나라부터 갈까?" (기능2 중심, 권역 병렬 분석)
```
S0 → S2(유럽 선택) → [분석 실행]
   → 상단 우측 "유럽 권역 분석 중 (3/7 국가)" 카드 표시
   → [상세보기 클릭] → 국가별 에이전트 진행 팝업
      • 체코: 100% ✓
      • 스페인: 100% ✓
      • 폴란드: 85% ⏳
      • 헝가리: 70% ⏳
      • 영국: 60% ⏳
      • 프랑스: 45% ⏳
      • 이탈리아: 30% ⏳
   → [약 2~3분 후 완료]
   → S2 순위표 자동 갱신 (1.체코 2.헝가리 ...)
   → 1위 행 클릭 → S1(체코 상세) → S4(보고서)
```

### 여정 C — "새 나라 조사부터" (데이터 우선)
```
S0 → S5(폴란드 조사 데이터 입력) → [조사완료]
   → "이 국가 진단하기" → S1(폴란드) 
   → [분석 실행] → [멀티 에이전트 분석]
   → S4
```

### 여정 D — "분석 중 다른 작업" (멀티태스킹)
```
S0 → S1(인도 입력) → [분석 실행]
   → 상단 카드 "인도 분석 중 45%" 표시
   → [분석 중 S3 이동] → 가중치 확인
   → [분석 중 S5 이동] → 다른 국가 데이터 입력
   → [상단 카드 "✓ 인도 분석 완료" 알림]
   → [카드 클릭] → "보고서 보기"
   → S4(인도 보고서)
```

## 4.5 멀티 에이전트 분석 아키텍처 (신규)

### 4.5.1 에이전트 구성 및 역할

| 에이전트 | 담당 카테고리 | 주요 리서치 항목 | 출력 |
|---|---|---|---|
| **Market Agent** | 시장 | 오토금융 침투율, 평균 차량가격, 개인:법인 비율, 딜러 집중도, 시장 규모 | 시장 점수 + 항목별 유사도 |
| **Regulation Agent** | 규제 | 외국인 지분 한도, 법인세율, 개인정보보호, 차량회수 소요기간, 라이선스 요건 | 규제 점수 + 킬스위치 검증 결과 |
| **Environment Agent** | 금융환경 | 평균 금리, 평균 LTV, 캡티브 강도, 신용평가 체계 | 환경 점수 + 항목별 유사도 |
| **System Agent** | 시스템 | 코어시스템 유형, 벤더 락인, 딜러 시스템 연동, 디지털 채널 성숙도 | 시스템 점수 + 게이트 검증 결과 |
| **Summary Agent** | 통합 | 4개 에이전트 결과 취합, 가중치 적용, 최종 판정, 비용 추정 | 종합 점수 + 판정 + 보고서 데이터 |

    0→100%    0→100%    0→100%    0→100%
         ↓         ↓         ↓         ↓

┌─────────────────────────────────────┐
│  Phase 2: 통합 & Summary (순차)     │

              ↓

## 5. 화면 간 공통 규칙

1. **조사일자 표시**: S1·S2·S4는 사용된 조사 데이터의 조사일자를 항상 노출(진출국 스냅샷 vs 대상국 현행 시점 격차).
2. **가중치 룰셋 표시**: 분석 결과 화면(S1·S2·S4)은 현재 적용 중인 ruleset 이름·버전을 항상 표시.
3. **킬스위치 우선 경고**: 판정이 BLOCKED(킬스위치 작동)면 점수와 무관하게 최상단 경고로 표시.
4. **공식 출처 갭 표시**: Tier2·3로 채운 항목은 결과·보고서에서 "공식 출처 미확보" 뱃지 노출.
5. **뒤로가기 일관성**: 모든 하위 화면은 S0로 복귁 가능. 분석 결과는 휘발되지 않고 저장(SIMILARITY_RESULT) 후 재진입 가능.
6. **분석 상태 가시성 (신규)**: 분석 실행 중 모든 화면에서 상단 우측 상태 카드가 표시되며, 사용자는 언제든 진행 상황 확인 가능.
7. **멀티 에이전트 독립성**: 각 카테고리 에이전트(시장, 규제, 환경, 시스템)는 독립적으로 작동하며, 하나가 실패해도 다른 에이전트는 계속 진행.
8. **백그라운드 분석**: 분석 실행 중에도 사용자는 다른 화면(S3, S5 등)으로 이동 가능하며, 분석은 백그라운드에서 계속 진행됨.
9. **실시간 진행률**: 에이전트별 진행률은 실시간(또는 5초 단위)으로 업데이트되며, 사용자는 각 단계별 상태 메시지 확인 가능.
10. **완료 알림**: 분석 완료 시 상태 카드가 "✓ 완료"로 전환되며, 사용자가 다른 화면에 있어도 알림 표시.

## 6. 멀티 에이전트 구현 참고사항

### 6.1 프론트엔드 구현

**상태 관리 (State Management)**
```javascript
{
  analysisStatus: {
    isRunning: boolean,
    country: string,
    overallProgress: number, // 0-100
    agents: {
      market: { progress: number, status: 'running'|'completed'|'error', message: string },
      regulation: { progress: number, status: 'running'|'completed'|'error', message: string },
      environment: { progress: number, status: 'running'|'completed'|'error', message: string },
      system: { progress: number, status: 'running'|'completed'|'error', message: string },
      summary: { progress: number, status: 'waiting'|'running'|'completed'|'error', message: string }
    },
    startTime: timestamp,
    estimatedCompletion: timestamp
  }
}
```

**WebSocket / Server-Sent Events (SSE)**
- 백엔드로부터 실시간 진행률 업데이트 수신
- 5초마다 폴링 또는 이벤트 기반 업데이트

**컴포넌트 구조**
```
<AnalysisStatusCard>           // 상단 우측 토글 카드
  <ProgressBar />              // 전체 진행률
  <QuickSummary />             // 에이전트별 간단 상태
  <ExpandButton />             // 상세 팝업 열기
</AnalysisStatusCard>

<AnalysisDetailModal>          // 상세 진행 팝업
  <OverallProgress />
  <AgentProgressList>
    <MarketAgent />
    <RegulationAgent />
    <EnvironmentAgent />
    <SystemAgent />
    <SummaryAgent />
  </AgentProgressList>
  <ViewReportButton />         // 완료 시 활성화
</AnalysisDetailModal>
```

### 6.2 백엔드 구현

**에이전트 실행 흐름**
```python
# Phase 1: 병렬 에이전트 실행
async def run_analysis(country_code, comparison_basis):
    # 4개 에이전트 병렬 실행
    results = await asyncio.gather(
        MarketAgent.analyze(country_code),
        RegulationAgent.analyze(country_code),
        EnvironmentAgent.analyze(country_code),
        SystemAgent.analyze(country_code)
    )
    
    # Phase 2: Summary 에이전트
    final_result = await SummaryAgent.synthesize(results)
    
    return final_result
```

**진행률 업데이트 API**
```
WebSocket: /ws/analysis/{analysis_id}
→ 메시지 형식:
{
  "type": "progress",
  "agent": "market" | "regulation" | "environment" | "system" | "summary",
  "progress": 0-100,
  "status": "running" | "completed" | "error",
  "message": "침투율 데이터 수집 중..."
}
```

### 6.3 에이전트 구현 가이드

각 에이전트는 다음 인터페이스를 구현:

```typescript
interface Agent {
  analyze(countryCode: string, comparisonCountry: string): Promise<AgentResult>;
  getProgress(): number;
  getStatus(): AgentStatus;
  cancel(): void;
}

interface AgentResult {
  category: string;
  score: number;
  items: Array<{
    name: string,
    comparisonValue: any,
    targetValue: any,
    similarity: number,
    source: string,
    tier: 'T1' | 'T2' | 'T3'
  }>;
  warnings: string[];
}
```
