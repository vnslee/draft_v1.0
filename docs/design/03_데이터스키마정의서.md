# 데이터 스키마 정의서

> 버전 v0.1 · 작성일 2026-06-14 · 용도: 에이전트 대시보드 DB 설계 기준
>
> 표기: PK=기본키, FK=외래키, ↳=참조. RDB 기준이되 NoSQL 전환 가능하도록 정규화.

---

## 0. 엔터티 관계 개요

```
COUNTRY ──┬─< ENTRY_RECORD          (진출국 실적, 1:1 또는 1:N)
          ├─< RESEARCH_SNAPSHOT ──< RESEARCH_ITEM   (조사 회차 → 항목)
          └─< SIMILARITY_RESULT ──< CATEGORY_SCORE  (분석 결과 → 카테고리 점수)

WEIGHT_RULESET ──< WEIGHT_ITEM       (가중치 룰셋 → 항목별 가중치)

SIMILARITY_RESULT ── REPORT          (결과 1:1 보고서)
SIMILARITY_RESULT ↳ WEIGHT_RULESET   (사용된 룰셋 스냅샷 참조)

CATALOG_CATEGORY ──< CATALOG_ITEM     (항목 마스터: 정의서의 항목 정의)
```

## 4. RESEARCH_ITEM — 조사 항목 값

회차 안의 개별 항목 데이터. 정의서의 각 항목이 여기에 한 행씩.

| 컬럼 | 타입 | 제약 | 설명 |
|---|---|---|---|
| item_id | string | PK | |
| snapshot_id | string | FK ↳RESEARCH_SNAPSHOT | |
| catalog_item_id | string | FK ↳CATALOG_ITEM | 항목 정의 참조 |
| value_raw | string | | 원본 값 |
| value_normalized | decimal/json | | 정규화 값(유사도 계산용). RANGE 타입은 {min, max} 형태 |
| source_name | string | | 출처 문서명/URL |
| source_type | enum | | LAW / FINANCIAL_STATEMENT / REGULATOR / ASSOCIATION / INTL_ORG / SECONDARY / FIELD |
| source_tier | enum | | TIER1(공식1차) / TIER2(준공식) / TIER3(참고) / BLOCKED(차단) |
| is_official | bool | | Tier1 여부(=공식 1차 출처). 보고서 인용 기준 |
| official_gap_flag | bool | | 공식 1차 미확보로 Tier2·3로 채운 경우 true |
| confidence_grade | enum | | OFFICIAL / SEMI_OFFICIAL / ESTIMATED (tier에서 자동 매핑) |
| survey_date | date | | 항목 단위 조사일(회차와 다를 수 있음) |

> **출처 등급제 (설계서 5.3 연동)**: 출처 판정은 **코드 레벨 도메인 화이트리스트**로 강제한다(LLM 프롬프트에 맡기지 않음). `BLOCKED`(뉴스·블로그·위키·익명)는 코드에서 원천 배제.
> - `source_tier` → `confidence_grade` 자동 매핑: TIER1→OFFICIAL(1.0), TIER2→SEMI_OFFICIAL(0.8), TIER3→ESTIMATED(0.5), BLOCKED→점수 제외.
> - **공백 방지 원칙**: 공식 1차(Tier1)를 못 찾아도 행을 비우지 않는다. Tier2·3로 채우고 `official_gap_flag=true`를 달아 신뢰도 계수로 자동 감점한다. 화이트리스트의 목적은 데이터를 막는 게 아니라 출처 약한 데이터를 점수에서 할인하는 것.
> - 화이트리스트는 **국가별로 다름** → 설정으로 관리, 점진 확장(로드맵 4번).

> - "가중치 저장" → weight_snapshot (ruleset 참조 + 전체 사본 이중 보관)

## 10. 핵심 무결성 규칙

1. **보고서 재현성**: REPORT는 ruleset_id 참조 + weight_snapshot 사본을 둘 다 보관. 룰셋이 삭제돼도 보고서는 자립.
2. **룰셋 불변성**: 보고서가 참조한 ruleset은 `is_locked=true`. 변경 시 새 버전 생성.
3. **출처 등급 게이트**: 출처 판정은 코드 레벨 화이트리스트로 강제(설계서 5.3). BLOCKED 출처는 점수에서 제외. Tier2·3는 신뢰도 계수로 자동 감점하되 데이터는 보존(공백 방지).
4. **조사일자 추적**: 모든 RESEARCH_ITEM은 survey_date 필수. 진출국 스냅샷(ENTRY_PREP)과 대상국(CURRENT)의 시점 격차를 보고서에 자동 표기.
5. **킬스위치 우선**: killswitch_hit이 있으면 total_score와 무관하게 verdict=BLOCKED.
