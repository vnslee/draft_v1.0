# 오토금융 해외진출 진단 콘솔 — 화면 요구사항 명세서

> 본 문서는 `오토금융진출진단 프로토타입.html`을 기준으로 작성한 화면 요구사항 명세서입니다.
> 프로토타입에 표시된 수치·국가·점수 등은 모두 **목(Mock) 데이터**이며, 본 명세서에는 실제 값이 아닌 **데이터 항목의 정의·성격**만 기술합니다.

---

## 1. 개요

### 1.1 시스템 목적
진출국(이미 진출 완료한 국가)의 실적 데이터를 기준으로, 미진출 대상국의 진출 가능성을 **유사도 기반으로 진단**하고, 진출 비용을 추정하며, 판정 근거 보고서를 생성하는 분석 콘솔이다.

### 1.2 핵심 진단 로직 (전 화면 공통 개념)
판정은 다음 순서로 수행된다.

1. **규제 킬스위치 검증** — 특정 규제 항목 불충족 시 점수와 무관하게 즉시 **진출 불가(차단)** 판정
2. **시스템 게이트 검증** — 시스템 유사도가 게이트 임계값 미만이면 코어 이식 불가
3. **종합 점수 산출** — 카테고리별 점수에 가중치·신뢰도 계수를 적용해 합산, 이식 임계값과 비교

판정 결과는 3단계로 구분된다.
- **이식 가능 (Transplantable)** — 임계값 이상, 킬스위치·게이트 통과
- **심층조사 (Investigate)** — 임계값 미달, 경쟁사·로컬사 추가 조사 권고
- **차단 (Blocked)** — 킬스위치 작동으로 진출 불가

### 1.3 화면 구성 (네비게이션)
| 코드 | 화면명 | 그룹 |
|------|--------|------|
| S0 | 대시보드 | 분석 |
| S1 | 단일 국가 진단 | 분석 |
| S2 | 권역 순위 | 분석 |
| S3 | 설정 (가중치 & 판정 기준) | 관리 |
| S5 | 데이터 관리 | 관리 |
| S4 | 보고서 뷰어 | 산출물 |

### 1.4 공통 레이아웃 요소
- **좌측 네비게이션 바**: 브랜드 영역 + 그룹별 메뉴 항목. 현재 화면이 활성 표시됨.
- **상단 바(Topbar)**:
  - **브레드크럼**: 현재 화면명 표시
  - **적용 룰셋 칩**: 현재 적용 중인 가중치 룰셋 버전 표시 (예: 디폴트 v1). 설정 저장 시 갱신됨.

## 2. S0 — 진출 진단 대시보드

### 2.1 목적
진출국과 미진출국 현황을 세계지도와 요약 지표로 한눈에 조망하고, 개별 국가로 진입하는 출발점.

### 2.2 화면 요소
| 요소 | 설명 |
|------|------|
| 요약 통계 카드 (4종) | 진출국 수, 분석 권역 수, 조사 완료 국가 수, 생성된 보고서 수 |
| 세계지도 (SVG) | 국가별 진단 상태를 색상으로 구분 표시 |
| 지도 범례 | 진출국 / 이식 가능 / 심층조사 / 차단(킬스위치) / 미조사 의 5개 색상 구분 |
| 국가 상세 카드 | 지도에서 국가 클릭 시 우측에 표시되는 요약 패널 |
| 최근 분석 목록 | 최근 진단한 국가 목록. 클릭 시 보고서로 이동 |

### 2.3 인터랙션
- 지도 위 국가에 **마우스 오버** 시 툴팁(국가명·점수 또는 진출국 표시).
- 국가 **클릭** 시 우측 상세 카드 갱신.
- 상세 카드 내 버튼은 국가 상태별로 분기:
  - 대상국(진단 가능) → "단일 진단 보기" (S1로 이동, 해당 국가 선택)
  - 진출국 → "실적 데이터 보기" (S5로 이동)
  - 기타 → "진단 실행" (S1로 이동)

### 2.4 데이터 항목 (지도/상세 카드)
국가별로 다음 속성을 가진다.
- **국가 식별 정보**: 국가 코드(ISO), 국가명, 국기
- **권역**: 소속 권역 (유럽 / 아시아·태평양 / 미주)
- **상태 구분**: 진출국 / 이식 가능 / 심층조사 / 차단 / 미조사
- **종합 유사도 점수**: 대상국에 한해 부여 (진출국은 점수 없음)
- **비고**: 비교국 또는 차단 사유(작동한 킬스위치명)
- **진출 비용**: 진출국에 한해 표시 (실적 데이터)
- **연결 코드**: 단일 진단(S1)으로 연결 가능 여부 식별자

## 3. S1 — 단일 국가 진단

### 3.1 목적
선택한 대상국을 같은 권역의 진출국과 비교해 유사도를 산출하고, 판정·비용·카테고리별 분석을 제공.

### 3.2 화면 요소
| 요소 | 설명 |
|------|------|
| 대상 국가 선택 | 미진출 대상국 드롭다운 |
| 비교 기준 선택 | "권역 내 최고 유사 진출국" / "전체 진출국 평균" 중 선택 |
| 진단 실행 버튼 | 선택 조건으로 진단 수행 |
| 판정 결과 영역 | 점수 링 + 판정 배지 + 비용 요약 |
| 카테고리별 유사도 막대 | 카테고리별 점수·가중치·게이트 시각화 |
| 경고 알림 영역 | 킬스위치 작동/게이트 미충족 시 표시 |

### 3.3 인터랙션
- 화면 최초 진입 시 결과가 없으면 자동으로 진단 실행.
- **킬스위치 작동** 시: 적색 경고 배너 표시, 비용 추정 보류("산출 안 함").
- **시스템 게이트 미충족** 시: 황색 경고 배너 표시.
- 결과 영역 하단 버튼: "근거 보고서 보기"(S4), "가중치 바꿔 재실행"(S3).

### 3.4 데이터 항목 (진단 결과)
대상국 단위로 다음 항목을 산출/표시한다.
- **종합 유사도 점수**: 0–100
- **판정 구분**: 이식 가능 / 심층조사 / 차단
- **비교국 정보**: 비교 기준이 된 진출국
- **카테고리별 점수**: 4개 카테고리(시장 / 규제 / 환경(금융) / 시스템) 각각의 유사도 점수 및 적용 가중치
- **시스템 게이트 충족 여부**: 시스템 카테고리에 게이트 임계선 표시
- **킬스위치 작동 여부**: 작동 시 차단 판정
- **비용 정보**: 비교국 대비 진출비용 비율(%), 추정 진출비용(USD), 예상 절감률(%)
  - ※ 킬스위치 작동 국가는 비용 미산출
- **시점 안내**: 비교국은 과거(진출준비시점) 스냅샷, 대상국은 현행 기준임을 명시

## 4. S2 — 권역 진출 우선순위

### 4.1 목적
선택한 권역의 미진출 국가 전체를 진단해 진출 점수 순으로 정렬한 순위표 제공.

### 4.2 화면 요소
| 요소 | 설명 |
|------|------|
| 분석 권역 선택 | 유럽 / 아시아·태평양 / 미주 드롭다운 |
| 순위 산출 버튼 | 선택 권역 진단 실행 |
| 순위 테이블 | 점수 내림차순 정렬된 국가 목록 |
| 각주 | 점수가 현재 적용 가중치 룰셋 기준임을 안내 |

### 4.3 인터랙션
- 권역 선택(onchange) 또는 버튼 클릭 시 순위 재산출.
- 화면 최초 진입 시 자동 산출.
- 테이블 **행 클릭** 시 해당 국가의 단일 진단(S1)으로 이동.

### 4.4 데이터 항목 (순위 테이블)
국가별 1행, 다음 컬럼으로 구성된다.
- **순위**: 점수 내림차순 순번
- **국가**: 국기 + 국가명
- **종합점수**: 0–100 (점수 구간별 색상 구분 — 고/중/저)
- **카테고리 미니 바**: 시장·규제·환경·시스템 4개 카테고리 점수의 막대 시각화 (구간별 색상)
- **판정**: 이식 가능 / 심층조사 / 차단 배지

## 5. S3 — 설정 (가중치 & 판정 기준)

### 5.1 목적
카테고리 가중치, 임계값, 신뢰도 계수, 규제 킬스위치를 조정하고 새 룰셋 버전으로 저장.

### 5.2 화면 요소
| 요소 | 설명 |
|------|------|
| 카테고리 가중치 슬라이더 (4종) | 시장 / 규제 / 환경(금융) / 시스템. 합계 100% 검증 |
| 가중치 합계 표시 | 합계 100% 여부 실시간 검증·표시 |
| 임계값 슬라이더 (2종) | 이식 임계, 시스템 게이트 |
| 출처 신뢰도 계수 (3종) | Tier1(고정 1.0) / Tier2 / Tier3 |
| 규제 킬스위치 토글 (5종) | 각 규제 항목 활성/비활성 |
| 저장 버튼 | "새 룰셋으로 저장", "저장 후 재분석"(S1로 이동) |

### 5.3 인터랙션
- 가중치 슬라이더 조정 시 **합계 검증**: 100%면 정상(✓), 아니면 "100%로 맞춰주세요" 경고.
- 임계값/계수 슬라이더 조정 시 수치 실시간 갱신.
- 킬스위치 토글 클릭 시 활성/비활성 전환.
- 저장 시 **새 룰셋 버전 생성**(예: 커스텀 v2), 상단 룰셋 칩 갱신. **기존 보고서는 생성 당시 가중치로 보존**(재현성 보장).

### 5.4 데이터 항목 (설정 값)
- **카테고리 가중치**: 4개 카테고리별 백분율 (합계 100%)
- **이식 임계값**: 이식 가능 판정 기준 점수
- **시스템 게이트 임계값**: 코어 이식 가능 최소 시스템 점수
- **출처 신뢰도 계수**: 출처 등급(Tier1/2/3)별 점수 보정 계수
  - Tier1(공식)은 1.0 고정, Tier2(준공식)·Tier3(참고)는 조정 가능
- **규제 킬스위치 항목**: 항목명 + 설명 + 활성 여부. 항목 예시 성격:
  - 외국인 지분 한도 — 사업모델 허용 가능한 지분 제한 여부
  - 외환 송금 규제 — 본사 자금 회수 가능 여부
  - 배당 송금 제한 — 이익 본국 송환 제한 여부
  - 데이터 현지화 의무 — 시스템 구축 비용 직결
  - 금리 상한(우수라) — 사업 수익성 직결
- **룰셋 버전**: 저장 시마다 증가하는 버전 식별자

## 6. S4 — 근거 보고서 (보고서 뷰어)

### 6.1 목적
단일 진단 결과의 판정 근거를 9개 섹션으로 상세 제시하는 산출물 문서. 생성 시점 룰셋으로 고정(재현성 보장).

### 6.2 화면 요소
- **보고서 헤더**: 보고서 ID, 대상국명, 권역, 비교국, 대상국 조사일, 비교국 스냅샷 시점, 적용 룰셋
- **섹션 내비게이션**: 9개 섹션 앵커 링크
- **상단 액션 버튼**: "원본 분석으로"(S1), "이 보고서 가중치로 재실행"(S3)

### 6.3 보고서 섹션 구성 (9개)
| # | 섹션 | 데이터 항목 설명 |
|---|------|------------------|
| 1 | 경영 요약 | 종합 점수, 판정 결과, 킬스위치/게이트 통과 여부, 추정 비용 및 절감률, 데이터 신뢰도 등급 요약 |
| 2 | AI 인사이트 | 핵심 판단 / 주의 지점 / 시점 리스크에 대한 서술형 분석 |
| 3 | 판정 근거 (점수 산출) | 카테고리 점수 × 가중치 합산식, 신뢰도 계수 적용, 임계값 비교 결과 |
| 4 | 킬스위치 & 게이트 검증 로그 | STEP별 검증 과정: ①킬스위치 항목별 PASS/FAIL ②시스템 게이트 ③종합 점수 임계 비교 → 최종 판정 |
| 5 | 카테고리별 항목 분석 | 4개 카테고리 각각, 진출국(비교국) vs 대상국의 **항목별 값 비교표**와 항목 유사도. 접기/펼치기(details) |
| 6 | 진출 비용 추정 | 진출국 실적 비용을 항목별로 분해, 유사도 기반 이식/재구축 비율 적용, 대상국 추정액·근거 |
| 7 | 리스크 & 검토 필요 항목 | 리스크별 심각도(HIGH/MED/LOW) + 설명 |
| 8 | 출처 목록 (등급별) | 항목별 출처와 등급(T1/T2/T3), "공식 미확보" 표시 여부 |
| 9 | 방법론 & 가중치 스냅샷 | 적용 룰셋, 카테고리 가중치, 임계값, 신뢰도 계수, 활성 킬스위치 — 생성 시점 고정 |

### 6.4 데이터 항목 상세

**6.4.1 카테고리별 항목 분석 (섹션 5)**
각 카테고리는 다수의 비교 항목으로 구성되며, 항목별로 다음을 가진다.
- 항목명
- 비교국(진출국)의 값 — 과거 스냅샷 기준
- 대상국의 값 — 현행 기준
- 항목 유사도 점수 (구간별 색상, "일치" 표기 가능)

카테고리별 대표 항목 성격(예시):
- **시장**: 오토금융 침투율, 평균 차량가격, 개인:법인 비율, 딜러 집중도
- **규제**: 외국인 지분 한도, 법인세율, 개인정보보호 체계, 차량회수 소요기간
- **환경(금융)**: 평균 금리, 평균 LTV, 캡티브 강도
- **시스템**: 코어시스템 유형, 벤더 락인, 딜러 시스템 연동, 디지털 채널 성숙도

**6.4.2 진출 비용 추정 (섹션 6)**
비용 항목별로 다음을 가진다.
- 비용 항목명 (라이선스/인허가, 법인 설립, 시스템 구축, 현지 자문, 인력/기타 등)
- 진출국 실적 비용
- 적용 비율(이식/재구축 비율)
- 대상국 추정 비용
- 산정 근거
- ※ 표본 수가 적어 오차 범위가 존재함을 명시

**6.4.3 출처 등급 (섹션 8)**
- 출처는 도메인 화이트리스트 기준으로 등급 판정
- **T1(공식 1차)**: 신뢰도 계수 1.0
- **T2(준공식)**, **T3(참고)**: 신뢰도 계수 적용, "공식 미확보" 표시

## 7. S5 — 데이터 관리

### 7.1 목적
진출국 실적 데이터(기준)와 대상국 조사 데이터(스냅샷)를 관리.

### 7.2 화면 요소
| 요소 | 설명 |
|------|------|
| 진출국 실적 테이블 | 진출 완료국의 기준 데이터 |
| 대상국 조사 데이터 테이블 | 미진출 대상국의 조사 현황. 행별 "진단하기" 버튼(S1 이동) |
| 각주 | 공식출처 비율(T1)이 낮으면 신뢰도 계수가 자동 하향됨을 안내 |

### 7.3 데이터 항목

**7.3.1 진출국 실적 (기준 데이터)**
- 국가 (국기 + 국가명)
- 권역
- 상태 (진출)
- 진출 시점 (연-월)
- 준비 기간 (일수)
- 진출 비용 (USD)
- 코어시스템 유형 (패키지/SI 등)

**7.3.2 대상국 조사 데이터**
- 국가 (국기 + 국가명)
- 권역
- 최근 조사일
- 데이터 구분 (현행 등)
- 충진율 (데이터 채움 비율 %)
- 공식출처 비율 (T1 비중, 등급 태그)
- 액션 (진단하기 → S1)

### 7.4 인터랙션
- 조사 데이터는 갱신 시마다 **조사일자와 함께 새 스냅샷으로 저장**.
- 공식출처 비율이 낮은 국가는 진단 시 신뢰도 계수가 자동 하향(조사 보강 권장).

## 8. 공통 데이터 개념 정리

| 개념 | 설명 |
|------|------|
| **진출국** | 이미 진출 완료한 국가. 비교 기준이 되는 실적 데이터 보유. 점수 없음. |
| **대상국** | 미진출 국가. 진단·점수·판정 대상. 현행 조사 데이터 보유. |
| **권역** | 국가 분류 단위 (유럽 / 아시아·태평양 / 미주). 진단 시 같은 권역 내 진출국과 비교. |
| **카테고리** | 유사도 평가 4개 축: 시장 / 규제 / 환경(금융) / 시스템. |
| **종합 유사도 점수** | 카테고리 점수 × 가중치 × 신뢰도 계수의 가중 합산. 0–100. |
| **이식 임계값** | 이식 가능 판정 기준 점수. |
| **시스템 게이트** | 코어 시스템 이식 가능 최소 시스템 점수. 미충족 시 점수 무관 이식 불가. |
| **규제 킬스위치** | 충족 시 점수 무관 즉시 차단되는 규제 항목. |
| **신뢰도 계수** | 출처 등급(T1/T2/T3)별 점수 보정 계수. |
| **룰셋 버전** | 가중치·임계값·킬스위치 설정의 버전. 보고서는 생성 시점 룰셋으로 고정(재현성). |
| **스냅샷 시점 격차** | 비교국(과거)과 대상국(현행) 기준 시점 차이. 보고서에 리스크로 명시. |

## 10. 상태·색상 규칙 (Status & Color Mapping)

판정·점수·등급은 일관된 색상 체계로 표현된다. (의미 전달이 색상에만 의존하지 않도록 텍스트 라벨/배지를 함께 표기)

### 10.1 판정 상태별 색상
| 상태 | 의미 | 색상 계열 |
|------|------|-----------|
| 이식 가능 | 임계값 이상 통과 | 그린(teal-green) |
| 심층조사 | 임계값 미달, 추가 조사 권고 | 앰버(amber) |
| 차단 | 킬스위치 작동, 진출 불가 | 테라코타(적색) |
| 진출국 | 기준 데이터 보유국 | 잉크(다크) |
| 미조사 | 조사 데이터 없음 | 그레이 |

### 10.2 점수 구간별 색상
점수·유사도 막대는 구간에 따라 색을 달리한다.
- **70점 이상**: 그린 (양호)
- **50–69점**: 앰버 (주의)
- **50점 미만**: 적색 (미흡)

### 10.3 출처 등급 태그
- **T1** (공식 1차): 그린 태그
- **T2** (준공식): 앰버 태그
- **T3** (참고): 골드 태그
- **공식 미확보**: 적색 보조 태그(T2·T3 항목에 병기)

### 10.4 리스크 심각도
- **HIGH**: 적색 / **MED**: 앰버 / **LOW**: 그린

## 11. 비기능 요구사항 (Non-Functional Requirements)

### 11.1 반응형 / 레이아웃
- 기본 레이아웃은 좌측 고정 네비게이션 + 우측 본문(최대 너비 약 1180px, 보고서는 약 860px).
- **화면 폭 860px 이하(모바일/협소)**:
  - 좌측 네비게이션 숨김.
  - 2단/3단 그리드(런처·비교·지도 등)는 1단으로 접힘.
  - 통계 카드는 2열로 재배치.
  - 판정 영역(점수 링 + 본문)은 세로 정렬로 전환.
- 세계지도(SVG)는 컨테이너 폭에 맞춰 비율 유지 스케일링.

### 11.2 재현성 (Reproducibility)
- 보고서는 **생성 시점의 룰셋(가중치·임계값·계수·킬스위치)으로 고정**되어, 이후 설정 변경과 무관하게 동일한 점수·판정을 재현한다.
- 설정 저장 시 기존 룰셋을 변경하지 않고 **새 버전을 생성**한다(이력 보존).

### 11.3 데이터 시점 정합성
- 비교국 데이터는 **진출준비시점(과거) 스냅샷**, 대상국 데이터는 **현행 기준**으로 분리 관리하며, 화면·보고서에 각 시점을 명시한다.
- 대상국 조사 데이터는 갱신 시마다 **조사일자를 포함한 새 스냅샷**으로 저장한다.

### 11.4 데이터 신뢰도 표현
- 진단 결과·보고서에 **데이터 신뢰도 등급**(예: A/B/C)과 공식출처(T1) 비율을 함께 제시한다.
- 공식출처 비율이 낮은 국가는 **신뢰도 계수가 자동 하향**되고, 사용자에게 조사 보강을 권고한다.

### 11.5 일관성·접근성
- 모든 판정·점수·등급은 §10의 색상 규칙을 일관되게 적용한다.
- 색상과 함께 텍스트 라벨/배지를 병기해 색각 이상 사용자도 식별 가능하게 한다.
- 상단 룰셋 칩은 전 화면에서 현재 적용 룰셋을 상시 노출한다.
